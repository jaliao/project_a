## ADDED Requirements

### Requirement: 會員學習階層樹狀顯示
系統 SHALL 在 `/admin/members/[id]` 頁面提供「學習階層」分頁，以樹狀結構顯示指定會員在「啟動靈人」（courseCatalogId = 1）課程中的師生傳承關係。樹狀結構包含：向上一層老師（帶領該會員完成啟動靈人的老師）、向下 N 層學生（此會員帶領完成的學員及其後代），N 由管理員設定控制，預設為 3。只顯示 `graduatedAt IS NOT NULL` 的結業學員。

#### Scenario: 顯示老師節點
- **WHEN** 管理者進入會員詳情頁的「學習階層」分頁
- **THEN** 頁面顯示帶領該會員完成啟動靈人課程的老師姓名（`realName`）與靈人編號（`spiritId`），若無老師則顯示「無老師紀錄」

#### Scenario: 顯示學生清單（第一層）
- **WHEN** 管理者進入會員詳情頁的「學習階層」分頁
- **THEN** 頁面顯示此會員直接帶領完成啟動靈人的學生清單，每筆顯示姓名與靈人編號

#### Scenario: 顯示多層學生（N 層展開）
- **WHEN** 學生清單中的學員本身也帶領過學生，且目前深度 < N
- **THEN** 頁面縮排顯示下一層學生清單，遞迴展開直到第 N 層

#### Scenario: 無學生紀錄
- **WHEN** 此會員尚未帶領任何學員完成啟動靈人課程
- **THEN** 頁面顯示「此會員尚未帶領學員」

#### Scenario: 中心節點高亮
- **WHEN** 管理者進入學習階層分頁
- **THEN** 當前會員在樹狀圖中以視覺區隔顯示（例如粗體或不同背景色），與其他節點有明顯差異

#### Scenario: 節點連結至詳情頁
- **WHEN** 管理者點擊階層樹中任一學員節點
- **THEN** 跳轉至該學員的 `/admin/members/[id]` 詳情頁
